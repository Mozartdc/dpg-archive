디멘션 트리 뷰는 Synchron Player의 중심 화면입임. 세로 한 줄이 하나의 디멘션이며, 각 디멘션에는 여러 슬롯이 들어 있고, 흰 테두리는 현재 선택된 슬롯임. 각 디멘션에서 선택된 슬롯을 왼쪽부터 차례로 보면, 지금 어떤 아티큘레이션 조합을 쓰고 있는지 알 수 있음.

아직 샘플을 불러오지 않은 슬롯은 회색으로 표시되고, 슬롯 이름 왼쪽 칸을 더블클릭해서 불러오면 활성화됨.



<img src="/images/06.-Dimension-Tree-View-1.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />


### <strong>Switch Controls</strong>



<img src="/images/06.-Dimension-Tree-View-2.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />

각 디멘션은 세 가지 방식으로 전환할 수 있음.

- None이면 MIDI로 전환하지 않음.

- Note, White Note, Program Change, Sequence, Interval에서는 노드를 단계별로 선택함.

- Dim.Ctrl에서는 노드를 연속으로 전환하거나 크로스페이드함.

<strong>Sequence</strong>

Sequence를 쓰면 노드가 차례대로 바뀜. 옆의 두 값은 리셋 기준과 코드 묶음 기준임.



<img src="/images/06.-Dimension-Tree-View-3.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />

왼쪽 값은 리셋 기준임. 한동안 새 노트가 안 들어오면 처음으로 돌아감.

오른쪽 값은 코드로 묶는 기준임. 아주 짧은 간격으로 들어온 노트는 다음 순서로 넘기지 않고 한 덩어리로 처리함.

<strong>Interval</strong>

가운데 슬롯을 기준으로, 레가토로 이어진 음 사이 간격에 따라 슬롯이 바뀜. 새로 시작하는 음은 항상 가운데 슬롯에서 시작하고, 그다음부터는 겹쳐 들어오는 음의 간격에 맞춰 다른 슬롯으로 넘어감.


### <strong>Dimension Controllers </strong>

디멘션마다 조절 신호를 따로 붙여서 움직이게 할 수 있음. A부터 H까지 디멘션 컨트롤러는 총 8개이고, 이 가운데 하나를 어떤 디멘션에 지정하면, 그 디멘션 아래쪽에 해당 컨트롤 노브와 연결된 MIDI 컨트롤러가 표시됨.

그 상태에서 필드를 클릭하면, 그 디멘션 컨트롤에 어떤 입력 신호를 연결할지 고를 수 있음.

선택 가능한 항목

- Controller (MIDI CC)

- Pitch Bend

- Speed

- Velocity

- Aftertouch

- Aftertouch Release

이렇게 연결해 둔 컨트롤들은 PERFORM 탭에서도 한번에 확인할 수 있음. 트리 뷰 아래쪽을 보면 목록이 표시되고, 거기서 각 컨트롤러의 커브도 추가로 직접 만질 수 있음.



<img src="/images/06.-Dimension-Tree-View-4.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />


### <strong>Parallel Mode </strong>

패럴렐 모드는 디멘션 안의 노드를 부드럽게 넘기고 싶을 때 쓰는 기능임. 컨트롤러 옆에 있는 스택 아이콘으로 켜고 끌 수 있음. 켜진 상태는 파란색, 꺼진 상태는 회색으로 표시됨.
패럴렐 모드를 켜면 해당 디멘션의 모든 노드가 동시에 스트리밍되어 바로 재생에 사용할 수 있게 됨. 특히 연속적으로 값이 바뀌는 디멘션 컨트롤러와 같이 쓸 때 노드 사이 전환이 훨씬 자연스러워짐. 대신 CPU 사용량은 늘어남.
또 XFade와 Mix 두 가지 블렌드 모드는 패럴렐 모드가 켜져 있어야 선택할 수 있음.



<img src="/images/06.-Dimension-Tree-View-5.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />


### <strong>Slot Crossfade Editor</strong>

슬롯 크로스페이드 에디터는 디멘션 안의 슬롯들이 어떻게 넘어갈지를 직접 손보는 곳임.연필 아이콘을 누르면 이 에디터가 열림. 이미 값을 수정한 적이 있으면 아이콘에 파란 점이 함께 표시됨.

에디터 창에서는 디멘션에 들어 있는 각 슬롯이 한 줄씩 따로 표시됨. 지금 실제로 들리고 있는 슬롯은 해당 디멘션 색으로 강조되어 바로 구분할 수 있음.

화면에 보이는 그래프 모양은 어떤 스위치 컨트롤을 쓰고 있는지, 또 패럴렐 모드가 켜져 있는지에 따라 달라짐.

컨트롤이 None으로 되어 있거나, Note·White Note·Program Change·Sequence·Interval처럼 단계식으로 전환되는 스위치 컨트롤을 쓰는 경우에는 사각형 형태의 그래프가 표시됨.



<img src="/images/06.-Dimension-Tree-View-6.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />

연속형 디멘션 컨트롤러를 쓰는 경우에는, 각 슬롯이 어느 컨트롤 값 구간에서 활성화되는지가 그래프로 표시됨. 현재 들어와 있는 컨트롤 값은 파란색 세로 막대로 표시되어, 지금 어느 위치에 있는지 바로 확인할 수 있음.



<img src="/images/06.-Dimension-Tree-View-7.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />

패럴렐 모드가 켜져 있으면 슬롯끼리 크로스페이드할 수 있게 되고, 화면도 일반 구간 표시 대신 크로스페이드 커브 형태로 바뀜.
페이드가 어떻게 겹치고 넘어갈지는 흰색 핸들을 움직여서 조절하면 됨. 필요하면 마우스 보조 조작도 함께 쓸 수 있음.
창 위쪽의 Overlapping 또는 Minimum Overlapping 버튼을 누르면 기본 크로스페이드 형태를 바로 만들 수 있음.
여기서도 색이 들어간 행은 지금 실제로 활성화되어 있는 슬롯을 뜻함.



<img src="/images/06.-Dimension-Tree-View-8.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />

블렌드 모드를 MIX로 두면, 해당 슬롯은 항상 재생되는 상태가 됨. 이때도 그 슬롯 행에 색이 들어와서 현재 그렇게 동작하고 있다는 것을 표시해 줌.



<img src="/images/06.-Dimension-Tree-View-9.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />

스위치 컨트롤, 패럴렐 모드, 블렌드 모드를 어떻게 조합하느냐에 따라 디멘션 트리를 매우 폭넓게 활용할 수 있음.

현재 어떤 슬롯이 활성화되어 있는지는 Slot Crossfade Editor를 보면 바로 확인됨. 색상 표시와 노드 그래프를 통해 어떤 슬롯이 동작 중인지 한눈에 파악할 수 있음.

또 Slot Crossfade Editor 안의 여러 요소에 마우스를 올리면, Synchron Player 화면 오른쪽 아래에 관련 정보가 함께 표시되어 작업할 때 참고하기 좋음.

<mark class="notion-highlight-gray">Tip : Synchron FX Strings I에서는 패럴렐 모드를 꼭 필요할 때만 켜 두는 편이 좋음. 필요하지 않을 때 꺼 두면 스트리밍 부하를 크게 줄일 수 있음.</mark>


### <strong>Folder Slots and Patch Slots</strong>

슬롯은 역할에 따라 두 가지로 나뉨.

하나는 안에 다른 슬롯들을 더 담고 있는 폴더 슬롯이고, 다른 하나는 실제로 소리가 나는 아티큘레이션이 들어 있는 패치 슬롯임.

폴더 슬롯은 아래로 더 펼칠 수 있는 구조라서 작은 화살표로 구분됨. 반면 패치 슬롯은 실제로 들리는 연주 내용을 담고 있는 슬롯임.

이 둘은 서로 연결된 구조로 다뤄지기 때문에, 여기서는 폴더 슬롯을 부모 슬롯, 그 안에 들어 있는 패치 슬롯을 자식 슬롯이라고 부름.



<img src="/images/06.-Dimension-Tree-View-10.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />

디멘션에 슬롯을 더 넣고 싶으면, 마지막 슬롯 아래에 있는 + 버튼을 누르면 됨.

새 디멘션을 만들고 싶을 때는 패치 슬롯 위에 마우스를 올렸을 때 나타나는 + 버튼을 누르면 됨. 이렇게 하면 새 디멘션이 추가되면서, 원래 있던 패치 슬롯은 하위 슬롯을 담는 폴더 슬롯으로 바뀜.



<img src="/images/06.-Dimension-Tree-View-11.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />


### Enable/Disable Slots 

처음 프리셋을 불러오면 슬롯은 기본적으로 비활성화된 상태로 저장되어 있음. 그래서 처음에는 샘플이 바로 로드되지 않고 회색으로 표시됨.

어떤 슬롯의 샘플을 불러오려면, 이름 왼쪽에 있는 색 칸을 더블클릭하면 됨. 그러면 그 슬롯에 들어 있는 샘플과 그 아래의 자식 슬롯 샘플까지 함께 로드됨.

같은 작업은 슬롯 컨텍스트 메뉴에서도 할 수 있음. 슬롯을 오른쪽 클릭하면 해당 메뉴가 열림.



<img src="/images/06.-Dimension-Tree-View-12.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />



<img src="/images/06.-Dimension-Tree-View-13.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />

설정에서 아예 동작 방식을 바꿔 둘 수도 있음. Synchron Player Settings에 들어가면, 앞으로 불러오는 모든 프리셋과 패치에서 셀을 항상 강제로 활성화할지 정할 수 있음.


### Slot Context Menu 

슬롯을 오른쪽 클릭하면 슬롯 컨텍스트 메뉴가 열리고, 여기서 슬롯 관련 작업을 처리할 수 있음.

<strong>Copy slot</strong>은 선택한 슬롯과 그 안의 자식 슬롯 전체를 클립보드로 복사하는 기능임.

<strong>Paste slot</strong>은 클립보드에 들어 있는 슬롯과 그 자식 슬롯 전체를 현재 위치에 붙여넣는 기능임.

<strong>Delete</strong>는 선택한 슬롯과 그 아래 자식 슬롯까지 함께 삭제함.

<strong>Enable (number of affected slots)</strong>은 선택한 슬롯과 그 아래 자식 슬롯에 들어 있는 샘플을 로드함. 괄호 안 숫자는 이 작업에 포함되는 슬롯 수를 뜻함.

<strong>Use current folder for release</strong>는 현재 폴더에 들어 있는 릴리즈 샘플을 릴리즈용 샘플로 쓰게 하는 기능임.

<strong>Convert to patch slot</strong>은 폴더 슬롯에서만 보이는 항목이고, 선택한 폴더 슬롯을 패치 슬롯으로 바꿈.

<strong>Delete all child slots</strong>도 폴더 슬롯에서만 보이는 항목이고, 그 폴더 아래에 있는 자식 슬롯을 전부 삭제함.

<strong>Delete all child slots with missing patches</strong>는 선택한 폴더 슬롯 아래에서, 실제 패치가 들어 있지 않은 자식 슬롯만 골라서 삭제하는 기능임.


### <strong>Root Slot Context Menu</strong>



<img src="/images/06.-Dimension-Tree-View-14.png" alt="" style="max-width: 100%; height: auto; border-radius: 4px; display: block; margin: 10px 0;" />

이어서 트리 전체나 디멘션 단위로 처리하는 메뉴도 있음.

<strong>Change color</strong>는 선택한 디멘션의 색을 바꾸는 기능임.

<strong>Copy root slot</strong>은 전체 트리 구조를 담고 있는 루트 슬롯을 통째로 클립보드에 복사함.

<strong>Paste root slot</strong>은 클립보드에 들어 있는 루트 슬롯, 즉 전체 트리 구조를 그대로 붙여넣는 기능임.

<strong>Dump</strong>는 현재 선택되어 있는 슬롯 정보를 시퀀서로 보냄.

<strong>Reset instrument</strong>는 지금 열려 있는 악기 전체를 초기 상태로 되돌림.

<strong>Enable all slots</strong>는 모든 슬롯에 대해 샘플을 로드하도록 적용함.

<strong>Disable all slots</strong>는 모든 슬롯을 비활성화 상태로 돌림.

<strong>Delete all slots containing missing patches</strong>는 불러올 수 없는 패치가 들어 있거나, 해당 악기에서 사용할 수 없는 패치를 포함한 슬롯을 전부 삭제하는 기능임. 프리셋을 정리하거나 새로 만드는 작업에서 특히 유용함.
